# QuantMind user study — task script and questionnaires

Session length ~15–20 minutes. The facilitator reads the task prompts, does not
help beyond re-reading them, and records completion. Participants are randomly
assigned to build **A** (explanations shown) or build **B** (explanations hidden),
so any difference in trust can be attributed to the explanations rather than the
tool. Assignment and all responses are logged against the participant code only.

## Pre-session confidence (before using the tool)

On a scale of 1 (not at all) to 5 (completely):

- PRE1. How confident would you feel choosing how to split money across a few
  investments on your own?
- PRE2. How much would you trust an automated tool to suggest such a split?

## Tasks (facilitator records: completed unaided / completed with re-read / not completed)

1. **Create a portfolio.** "Open the dashboard and set up a portfolio using the
   five available companies." (Checks the core flow works for a non-expert.)
2. **Read a recommendation.** "Find what the system currently recommends holding
   the most, and how much of the portfolio that is." (Checks the recommendation is
   findable and legible.)
3. **Find the reason.** "Find the reason the system gives for that holding — why did
   it choose it?" (Build A: via the explanation. Build B: participants are asked the
   same question with no explanation available, to gauge the gap.)

For each task, record: completed (yes/re-read/no), time to complete, and any point
where the participant hesitated or asked for help.

## System Usability Scale (after using the tool)

Rate each 1 (strongly disagree) to 5 (strongly agree). The standard SUS is scored
0–100; benchmark: ~68 is average, above ~80 is good (Bangor et al., 2008).

1. I think I would like to use this system frequently.
2. I found the system unnecessarily complex.
3. I thought the system was easy to use.
4. I think I would need support from a technical person to use this system.
5. I found the various functions in this system were well integrated.
6. I thought there was too much inconsistency in this system.
7. I would imagine most people would learn to use this system very quickly.
8. I found the system very cumbersome to use.
9. I felt very confident using the system.
10. I needed to learn a lot of things before I could get going with this system.

## Post-session confidence and trust

On a scale of 1 to 5:

- POST1. After using the tool, how confident would you feel choosing an investment
  split? (compare with PRE1)
- POST2. How much did you trust the tool's recommendations? (compare with PRE2)
- POST3 (build A only). Did the reasons the tool gave make you more or less willing
  to act on its advice? (1 much less … 5 much more)
- Free text: one thing that helped you trust it, and one that did not.

## Debrief

Thank the participant, restate that the data is anonymised and deletable on
request, and answer any questions. Record the participant code, build (A/B), and
whether consent to use anonymised quotes was given.
